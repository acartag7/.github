# Contributing to Edictum

Thank you for your interest in contributing to Edictum! This document covers the guidelines for contributing to any repository in the edictum-ai organization.

## Contributor License Agreement (CLA)

Before we can accept your contribution, you must sign the Edictum Contributor License Agreement. This is a one-time requirement that ensures we can continue to distribute Edictum under its current and future licenses.

The CLA bot will automatically prompt you when you open your first pull request.

## Repositories

### edictum (MIT)

The core governance library. Contributions are welcome under the MIT license.

### edictum-console (FSL-1.0-Apache-2.0)

The operations console. Contributions are accepted under the FSL-1.0-Apache-2.0 license. By contributing, you agree that your contributions may be distributed under the FSL-1.0-Apache-2.0 license and, after the two-year change date, under Apache 2.0.

## How to Contribute

1. **Fork** the repository you want to contribute to
2. **Create a branch** from `main` with a descriptive name
3. **Make your changes** — follow the coding standards in each repo's `CLAUDE.md`
4. **Write tests** for any new functionality
5. **Open a pull request** with a clear description of your changes

## Reporting Issues

Please use the GitHub Issues tab on the relevant repository. Include:

- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Edictum version (`pip show edictum`)
- Python version and OS

## Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone. Please be respectful and constructive in all interactions.

## Questions?

Open a Discussion on the relevant repository or reach out at [edictum.ai](https://edictum.ai).
